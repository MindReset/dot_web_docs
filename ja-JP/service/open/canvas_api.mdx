## クイックスタート

### 必要条件

1. Dot. App で API キーを取得して保存します；
2. 所有しているデバイスのシリアル番号を取得します；
3. デバイスが電源とネットワークに接続されていることを確認します；
4. Dot. App のコンテンツ工房で `Canvas API` コンテンツをデバイスの**ループ**タスクに追加します。

<Callout type="info" title="Canvas API の用途">
	Text API はテキストカードをすばやく送る用途に向いており、Image API は 1 枚の画像表示に向いています。Canvas API では、自分のデータからカード、ダッシュボード、小さなビジュアル実験を作れます。
</Callout>

<Callout type="info" title="windowData はオブジェクト要素で画面を表します">
	`windowData` は React-elements-like objects に近い書き方です。各要素はオブジェクトで、`type` が要素種別、`props` がスタイルと内容、`props.children` がテキスト、子要素、または子要素の配列になります。
</Callout>

<Callout type="info" title="最外層コンテナには通常 padding は不要です">
	デバイスのレイアウトは `windowData.default` の外側で基本的な余白を処理します。最外層要素は構造、背景、整列を優先してください。全面表示、背景色変更、FULL レイアウトの上書きが必要な場合は `layoutFull` を渡します。
</Callout>

### リクエスト本文

Canvas API のリクエストは、内容データと画面構造を組み合わせて送信します。`data` には画面から読み取る値を入れ、`windowData` には要素構造を記述します。必要に応じて、`layoutFull` で FULL レイアウトを調整し、`link` でタップリンクを設定し、`border` で画面の枠色を選択できます。

フィールドの意味が衝突しないように、`data` の中では `type`、`key`、`windowData`、`layoutFull`、`link`、`border`、`__proto__`、`constructor`、`prototype` をキーとして使用しないでください。

```json
{
	"refreshNow": true,
	"data": {
		"title": "Canvas API",
		"message": "Hello Dot."
	},
	"windowData": {
		"default": [
			{
				"type": "div",
				"props": {
					"tw": "flex flex-1 bg-white text-black",
					"children": "{{get inputData \"title\"}}"
				}
			}
		]
	},
	"layoutFull": {
		"tw": "p-0 bg-white",
		"style": {
			"padding": 0
		}
	},
	"link": "https://dot.mindreset.tech",
	"border": 0
}
```

### windowData の書き方

- `windowData` には `default` レイヤー配列が必要です；
- 要素は通常 `{ "type": "div", "props": { ... } }` の形で書きます；
- `props.tw` には Tailwind-like なスタイル、`props.style` には明示的なインラインスタイルを書きます；
- `props.children` は文字列、1 つの要素オブジェクト、または要素配列を指定できます；
- 動的な内容は `{{get inputData "user.name" default="-"}}` のように、リクエスト `data` の値を読み取れます；
- よく使う要素は `div`、`span`、`img` です；
- 画像には data URI または匿名アクセス可能な `http(s)` 画像 URL を使用できます；
- 最外層コンテナに追加の padding は不要です。FULL レイアウトを上書きしたい場合は `layoutFull.tw` または `layoutFull.style` を使用します；
- 安定したレンダリングのため、リクエストのサイズと複雑さには制限があります。

### 対応しているスタイルとカスタム Tailwind クラス

Canvas API は、デバイス向けの静的レンダリングに適したスタイル能力を提供します。スタイルの境界は **React object-like 要素構造 + 静的 CSS サブセット**に、Dot. 独自の Tailwind フォント utility、ブレークポイント処理、画像処理クラスを足したものです。

| 書き方 | 対応元 | 説明 |
| --- | --- | --- |
| `props.style` | Dot Canvas 静的スタイルサブセット | ピクセル値や明示的なスタイルに向いています。 |
| `props.tw` | Tailwind-like 構文 + Dot. 拡張 | レイアウト、色、余白、フォントクラス、画像処理クラスを素早く書く用途に向いています。 |
| `layoutFull.tw` / `layoutFull.style` | Dot. layout 外枠の上書き | 全画面表示、背景、padding の上書きに使います。 |

Dot Canvas の `props.style` サポート一覧は、静的 CSS の対応表に合わせています。主なカテゴリには、CSS 変数、`display`、`position`、`color`、`margin`、`padding`、`top/right/bottom/left`、`width/height`、`minWidth/minHeight/maxWidth/maxHeight`、`border*`、`borderRadius*`、Flexbox、`gap`、`fontFamily`、`fontSize`、`fontWeight`、`fontStyle`、`tabSize`、`textAlign`、`textIndent`、`textTransform`、`textOverflow`、`textDecoration`、`textShadow`、`letterSpacing`、`lineHeight`、`whiteSpace`、`lineClamp`、`wordBreak`、`textWrap`、`backgroundColor`、`backgroundImage`、`backgroundPosition`、`backgroundSize`、`backgroundClip`、`backgroundRepeat`、`transform`、`transformOrigin`、`objectFit`、`objectPosition`、`opacity`、`boxSizing`、`boxShadow`、`overflow`、`filter`、`clipPath`、`mask*`、`WebkitTextStroke*` があります。

このスタイル一覧は `props.style` の範囲だけを示します。以下のフォントクラス、ブレークポイント処理、`img-*` 画像処理クラスは、Dot. が `props.tw` に追加しているカスタム Tailwind 拡張レイヤーです。

重要な制限：Canvas API は完全なブラウザ環境ではありません。`<style>`、外部 `<link>`、`<script>` は使えません。3D transform、`z-index`、`calc()` も対応していません。`currentColor` は `color` プロパティでのみ信頼できます。高度な組版機能や RTL 言語も現在の主な対象ではありません。Canvas API でも JSX、任意の JS 式、React hooks、ブラウザ API、外部 CSS、カスタムコンポーネントは使えません。

よく使う `props.tw` には、`flex`、`flex-row`、`flex-col`、`flex-1`、`shrink-0`、`grow`、`items-*`、`justify-*`、`w-full`、`h-full`、`w-[84px]`、`h-[40px]`、`min-w-0`、`min-h-0`、`max-h-[200px]`、`gap-*`、`gap-[5px]`、`p-*`、`px-[8px]`、`py-[5px]`、`bg-*`、`text-*`、`border*`、`rounded*`、`overflow-hidden`、`box-border`、`box-content`、`fill-black`、`fill-white` があります。Tailwind クラスが不確かな場合は `props.style` を優先してください。

文字には次のフォントクラスを使えます：

| 種類 | 構文 | 例 |
| --- | --- | --- |
| 通常フォント | `text-{size}-{font}` | `text-18-chillduansans` |
| 任意 px サイズ | `text-[Npx]-{font}` | `text-[20px]-playfairdisplay` |
| Tailwind サイズキー | `text-{xs/sm/base/lg/xl/2xl...}-{font}` | `text-lg-chillduansans` |
| ピクセルフォント | `text-pixel-{size}[-variant]` | `text-pixel-12-zpix` |

よく使う通常フォント key：`chillduansans`、`chillksans`、`chillorganic`、`chillroundf`、`chillroundgothic`、`logoscunboundedsans`、`maokenyingbikaishuj0.09`、`playfairdisplay`、`zihunzhoukesong`。

ピクセルフォントクラス：`text-pixel-8`、`text-pixel-8-quan`、`text-pixel-10`、`text-pixel-12`、`text-pixel-12-xiaoya`、`text-pixel-12-zpix`、`text-pixel-16`、`text-pixel-16-cusong`、`text-pixel-16-unifont`、`text-pixel-16-unifontmono`、`text-pixel-24`。

テキスト組版では、`props.style` に `fontSize`、`fontWeight`、`lineHeight`、`letterSpacing`、`whiteSpace`、`wordBreak`、`lineClamp`、`textOverflow`、`overflow` を使うのが安定しています。複数行本文の例：

```json
{
	"tw": "text-18-chillduansans",
	"style": {
		"lineHeight": "20px",
		"whiteSpace": "pre-wrap",
		"wordBreak": "break-word",
		"overflow": "hidden"
	}
}
```

画像要素では、`img.props.tw` に電子ペーパー向け画像処理クラスを追加できます：

```txt
img-dither-none
img-dither-diffusion
img-dither-ordered

img-kernel-threshold
img-kernel-atkinson
img-kernel-burkes
img-kernel-floyd-steinberg
img-kernel-sierra2
img-kernel-stucki
img-kernel-jarvis-judice-ninke
img-kernel-diffusion-row
img-kernel-diffusion-column
img-kernel-diffusion-2d

img-levels-2
img-levels-3
img-levels-4
img-levels-8
img-levels-16
```

画像の配置には `style.objectFit` と `style.objectPosition`、たとえば `contain` / `center bottom` を使ってください。`aspect-ratio`、CSS Grid、絶対配置、疑似クラス、メディアクエリ、外部 CSS、スクリプト、カスタムコンポーネントには依存しないでください。これらは Canvas API の安定サポート範囲ではありません。
