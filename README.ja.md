# Dot. ドキュメント

**言語:** [English](README.md) | [简体中文](README.zh-Hans-CN.md) | [日本語](README.ja.md)

[![License](https://img.shields.io/badge/license-CC%20BY--NC--SA%204.0-blue.svg)](LICENSE)

Dot. 製品の公式ドキュメントリポジトリへようこそ！このリポジトリには、Dot. App および Dot. シリーズデバイスの包括的なユーザーマニュアル、API ドキュメント、FAQ、セキュリティアドバイザリが含まれています。

## 📚 このリポジトリについて

これは [MindReset](https://www.mindreset.tech) の Dot. ブランドのドキュメントセンターで、以下の内容を含みます：

- **製品マニュアル**: 詳細なインストール、設定、使用方法ガイド
- **デバイスドキュメント**: Quote/0 などのデバイスの完全な操作説明
- **開発者ドキュメント**: オープン API と Studio 統合ガイド
- **よくある質問**: トラブルシューティングと問題解決方法
- **セキュリティアドバイザリ**: 製品セキュリティアップデートと脆弱性の開示
- **ファームウェア更新履歴**: バージョン履歴とアップデートノート

## 🌍 多言語サポート

ドキュメントは以下の言語で提供されています：

- **简体中文** (`zh-Hans-CN/`)
- **English** (`en-US/`)
- **日本語** (`ja-JP/`)

## 🤝 コントリビューション

ドキュメントの改善にコミュニティメンバーの参加を歓迎し、奨励しています！エラーや漏れを見つけた場合、またはコンテンツを改善したい場合は、以下の方法で参加してください：

### Issue の提出

ドキュメントに問題を見つけた場合や改善提案がある場合は、[Issue を作成](../../issues)してください。

### プルリクエストの提出

1. このリポジトリをフォーク
2. フィーチャーブランチを作成 (`git checkout -b feature/improve-docs`)
3. 変更をコミット (`git commit -m 'docs: Quote/0 ペアリング手順を更新'`)
4. ブランチにプッシュ (`git push origin feature/improve-docs`)
5. プルリクエストを作成

### コントリビューションガイドライン

コンテンツを提出する前に、以下の点にご注意ください：

- **一貫性を保つ**: 既存のドキュメントの形式とスタイルに従ってください
- **正確性**: 技術情報の正確性を確保してください
- **多言語**: 可能であれば、すべての言語バージョンを同時に更新してください
- **明確な表現**: 簡潔で理解しやすい言葉を使用してください
- **画像の最適化**: 画像を追加する場合は、ファイルサイズが適切であることを確認してください
