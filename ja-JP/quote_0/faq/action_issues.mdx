## なぜ App Clip / アプリへ遷移するのか

Quote/0 のかざす操作は、固定された Web ページを開くものではありません。コンテンツを処理するための入口を開く仕組みです。

デバイスに表示されている内容は変化し、内容の種類によって適した表示方法も異なります。そのため、Dot. アプリまたは App Clip が現在の内容を取得し、統一された権限確認を行い、適切な開き方を選択する必要があります。

### NFC は入口を起動し、内容そのものは保持しない

Quote/0 の右側の空白部分には NFC モジュールが内蔵されています。スマートフォンをデバイスに近づけると、NFC が固定された入口を起動し、Dot. アプリまたは App Clip を開きます。

この入口は現在の画面内容を直接保存しているわけではなく、最終的に Web ページ、テキスト、写真、第三者アプリのどれを開くかも決定しません。

### アプリが現在の内容を取得し、権限を確認する

かざす操作でアプリが起動すると、アプリは現在のデバイス画面に対応する内容を取得します。

これにより NFC の入口は安定したまま、実際に開く内容はデバイスの現在状態に合わせて変化できます。NFC タグ自体が変わらなくても、最終的に表示される内容はデバイスの表示内容に応じて更新できます。

内容を表示する前に、アプリは統一された権限確認を行います。この権限確認は特定の内容種類だけに対するものではなく、かざす操作全体の基本手順です。

権限確認を通過した後、アプリは内容の種類に応じて表示方法を選択します。

### 内容によって開き方が異なる

| 内容の種類 | 開き方 |
| --- | --- |
| ニュース・記事系コンテンツ | Web ページを開く |
| 第三者アプリ向けコンテンツ | 対応するアプリへ遷移する |
| API または通常のテキスト | プレーンテキストとして表示する |
| 構造化テキスト | Markdown として表示する |
| 画像コンテンツ | 写真として表示する |

NFC が固定された Web ページを直接開くだけでは、これらの内容形式に対応できず、開く前の統一された権限確認も行えません。

### この設計により低消費電力も維持できる

Quote/0 は NFC を低消費電力の起動入口として使い続けることができ、かざす操作のためだけにデバイスを常時給電または常時オンラインにする必要がありません。

内容の取得、権限確認、表示方法の選択はアプリ側で処理し、デバイス本体は電子ペーパー画面とバッテリーモードの低消費電力設計を維持できます。

### iPhone

「かざす」操作を試しても iPhone で App Clip が起動しない場合は、iPhone を一度ロックしてから再びロック解除し、iPhone を下から上へ動かすようにしてデバイスの右側に近づけてみてください。多くの場合、これで正常に起動します。

この現象は、iOS のシステム制限によるものです。同じ App Clip のソースに何度もタッチすると、iOS が後続の起動をブロックすることがあります。画面をロックした後、十分な動きで再度近づけると、iOS が App Clip を再び起動します。

中国本土版の iPhone をお使いの場合は、「設定」→「一般」で NFC が有効になっているかご確認ください。

### Android

Android 端末で正しく起動しない場合は、Dot. アプリがインストールされていること、NFC 機能が有効になっていること、そしてお使いの端末の NFC アンテナ位置を確認してください。

その NFC 領域をデバイス右側の空白部分に近づけると、コンテンツを起動できます。
