MindReset の Dot. Quote/0 電子ペーパーデバイス向け Home Assistant カスタム統合です。Home Assistant からデバイスのステータスを監視し、テキスト・画像・天気・カレンダー・システム状態コンテンツを Quote/0 に直接プッシュできます。

### 機能

#### センサー

- **電源状態** — 現在のデバイス電源モード（充電アクティブ、バッテリーアクティブ等）
- **バッテリー状態** — 充電状態
- **Wi-Fi 信号強度** — 信号強度（dBm）
- **ファームウェアバージョン** — 現在のファームウェア
- **最終レンダリング** — 最後の画面更新のタイムスタンプ
- **次回レンダリング（バッテリー/電源）** — 予定されている次回更新時刻
- **コンテンツタスク数** — コンテンツループ内のタスク数（完全なタスクリストは属性で確認可能）

#### バイナリセンサー

- **オンライン** — 接続状態

#### コントロール（デバイスページ）

- **次のコンテンツ** — コンテンツループの次のアイテムに切り替え
- **テキスト送信** — タイトル・メッセージ・署名フィールドを使ってテキストをプッシュ
- **画像送信** — 画像データフィールドを使って画像をプッシュ
- **画像データ** — Base64 エンコードされた PNG（296×152px）または絶対ファイルパス
- **ディザリングタイプ** — ディザリングアルゴリズムを選択（DIFFUSION、ORDERED、NONE）

#### サービスアクション

- `dot_quote0.send_text` — 完全なパラメーター制御によるテキストコンテンツのプッシュ
- `dot_quote0.send_image` — ディザリングと枠線オプション付きの画像コンテンツのプッシュ
- `dot_quote0.send_weather` — HA の weather エンティティから現在の天気と予報をプッシュ
- `dot_quote0.send_calendar` — HA の calendar エンティティから今後の予定をプッシュ
- `dot_quote0.send_system_status` — Home Assistant のシステム状態（バージョン、CPU、メモリ、ディスク、エンティティ数）をプッシュ

#### 診断

- Home Assistant からデバイス診断情報をダウンロードし、接続やレンダリングの問題を切り分け可能

### HA-dot-quote0-card との連携

- **役割分担**: `HA-dot-quote0` はデバイス統合・エンティティ・サービスアクションを担当し、`HA-dot-quote0-card` はダッシュボード UI を担当
- **導入順序**: 先に本統合を導入し、その後カードを導入。カードは Home Assistant 上で検出済みの Quote/0 デバイスを利用
- **推奨運用**: 日常の手動操作とプレビューはカード、天気・カレンダー・システム状態などの定期実行は自動化でサービスアクションを利用

カードのページ: [/docs/service/co_create/software/ha_dot_quote0_card](/docs/service/co_create/software/ha_dot_quote0_card)

### 前提条件

1. Wi-Fi 接続済みでペアリング済みの Quote/0 デバイス
2. Dot App で発行した API キー
3. デバイスのシリアル番号

### インストール

#### HACS（推奨）

1. HACS → インテグレーション → ⋮ メニュー → カスタムリポジトリ を開く
2. `https://github.com/Never2land/HA-dot-quote0` をカテゴリ Integration で追加
3. Dot. Quote/0 をインストール
4. Home Assistant を再起動

#### 手動

`custom_components/dot_quote0/` フォルダーを Home Assistant の `config/custom_components/` ディレクトリにコピーして再起動します。

### 設定

1. 設定 → デバイスとサービス → 統合を追加 に移動
2. Dot. Quote/0 を検索
3. API キーを入力
4. すべての Quote/0 デバイスが自動的に検出されます

### 自動化例

サービスアクションで天気コンテンツをデバイスへ送信:

```yaml
service: dot_quote0.send_weather
data:
	serial: "YOUR_DEVICE_SERIAL"
	weather_entity: weather.home
	include_forecast: true
	forecast_days: 3
	refresh_now: true
```
