## 为什么碰一碰需要跳转轻 App / App

Quote/0 的「碰一碰」不是打开一个固定网页，而是打开一个内容处理入口。

因为设备当前显示的内容会变化，且不同内容需要不同的展示方式，所以需要由 Dot App 或轻 App 获取当前内容、完成统一权限判断，并选择合适的打开方式。

### NFC 负责唤起入口，不负责承载内容

Quote/0 右侧区域内置 NFC 模块。手机靠近设备后，NFC 会触发一个固定入口，用于唤起 Dot App 或轻 App。

这个入口本身不直接保存当前屏幕内容，也不决定最终打开网页、文本、照片还是第三方 App。

### App 负责获取当前内容和判断权限

碰一碰唤起 App 后，App 会根据设备信息获取当前屏幕对应的内容。

这一步可以保证 NFC 入口保持稳定，而实际打开的内容可以随着设备当前显示内容变化。即使 NFC 标签本身没有变化，最终展示的内容也可以根据设备当前状态更新。

在展示内容之前，App 会先进行统一权限判断。权限判断不是针对某一种内容单独存在，而是碰一碰流程中的基础步骤。

权限判断通过后，App 才会继续根据内容类型选择展示方式。

### 不同内容需要不同打开方式

| 内容类型 | 打开方式 |
| --- | --- |
| 资讯类内容 | 打开网页 |
| 第三方 App 内容 | 跳转到对应 App |
| API 或普通文本 | 以纯文本方式展示 |
| 结构化文本 | 以 Markdown 方式展示 |
| 图像内容 | 以照片方式展示 |

如果 NFC 直接打开固定网页，就无法同时适配这些内容形态，也无法在打开前完成统一权限判断。

### 这样设计也能保持低功耗

Quote/0 仍然可以把 NFC 作为低功耗触发入口，而不是让设备为了碰一碰持续供电或持续在线。

内容获取、权限判断和展示方式选择都交给 App 处理，设备本身可以继续保持墨水屏和电池模式下的低功耗设计。

## 为什么打不开轻 App / App

<CustomCloudImage osSrc="dot/docs/action_location.png" alt="“唤起位置" />

### iPhone

如果您在尝试「碰一碰」，却在 iPhone 中无法唤起轻 App，建议尝试将 iPhone 锁屏，重新解锁设备，手持 iPhone 由低到高，靠近设备右侧，这样一般可以正常唤起轻 App。

出现这种问题的原因，一般是由于 iOS 的系统限制。当您触碰多次相同的轻 App 来源时，iOS 会屏蔽后续的唤起操作，只有您锁屏后，并且动作幅度较大来去靠近时，iOS 才会重新唤起轻 App。

如若您正在使用国行 iPhone，可以检查「设置」 - 「通用」中的 NFC 是否已开启。

### Android

如果您在尝试「碰一碰」，却在 Android 设备中无法正确跳转，请确保您的 Android 设备已安装 Dot. App，开启 NFC 功能、并确定您的 Android 设备 NFC 区域。

将该区域靠近设备右侧空白区域，即可唤起内容。
