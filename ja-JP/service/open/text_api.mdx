## クイックガイド

### 必要条件

1. Dot. アプリで<Link href="/docs/service/studio/api/get_api">API キーを取得して保存</Link>していること
2. 自分のデバイスが少なくとも 1 台あり、<Link href="/docs/service/studio/api/get_device_id">デバイスのシリアル番号</Link>を取得していること
3. デバイスが電源に接続されていること
4. デバイスがネットワークに接続されていること
5. Dot. アプリの創造所から「テキスト API」コンテンツをデバイスの **ループ** タスクに追加していること

<Callout type="info" title="Text API の用途">
	Text API は、リマインダー、メモ、署名、小さなアイコンを読みやすいデバイスカードにします。自動化メッセージ、軽い通知、毎日の抜粋、すぐに見たい短いテキストに向いています。
</Callout>

<Callout type="info" title="API の表示タイミングについて">
	API の即時表示は、デバイスが現在オンラインかどうかに依存します。デバイスがバッテリー駆動でスリープ中の場合、内容はまずサーバーに保存され、デバイスが次回自動起動したあとに表示されます。詳しくは<Link href="/docs/quote_0/faq/refresh_timing">時間の仕組み</Link>をご覧ください。
</Callout>

<Callout type="info" title="アイコン入力について">
	`icon` フィールドには、PNG Base64 データまたは完全なオンライン `http(s)` 画像 URL を指定できます。リモートアイコン URL は匿名で直接アクセスでき、認証や特別な Referer を必要とせず、`image/*` を返す必要があります。
</Callout>

<Callout type="info" title="デバッグ用のテストアイコン URL">
	以下の URL は現時点で匿名アクセス可能で、Text API の `icon` フィールドに利用できる画像コンテンツを返します。

	- `https://placehold.co/40x40/png`
	- `https://httpbin.org/image/png`

	自分のアイコン URL が使えるか先に確認したい場合は、次を実行してください。

	```bash
	curl -I 'https://your-icon-url'
	```

	通常は少なくとも次の 2 点を確認してください。

	- HTTP ステータスコードが `200`
	- `Content-Type` が `image/*` であること。例: `image/png`、`image/jpeg`
</Callout>

### 改行とタブ

Text API は `message` 内の改行とタブを保持します。

- JSON リクエスト本文では、改行に `\n`、タブに `\t` を使います。
- JSON が shell、JavaScript、Python、テンプレート、ツール設定など別の文字列リテラルの中に入る場合、最終的な JSON に `\n` と `\t` を残すために、外側では `\\n` と `\\t` と書く必要があることがよくあります。
- デバイス上に文字どおり `\n` や `\t` を表示したい場合は、最終的な JSON ペイロードに `\\n` や `\\t` を含めてください。

```json
{ "message": "1行目\n\tインデントした2行目" }
{ "message": "文字どおりの \\n と \\t を表示" }
```

### レスポンスコード

| code  | 意味             | 説明                                                |
| :---- | :--------------- | :-------------------------------------------------- |
| `200` | 成功             | テキスト API コンテンツを切替／データ更新（未切替） |
| `400` | パラメータ不正   | 無効なアイコン形式                                  |
| `403` | 権限不足         | 当該デバイスを操作する権限がありません              |
| `404` | 未検出           | デバイスが存在しない・未登録／テキスト API コンテンツがループタスクに未追加   |
| `500` | デバイス応答失敗 | テキスト API コンテンツの切替に失敗                 |
