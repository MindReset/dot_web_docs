## Dot Skill とは？

Dot Skill は、AI アシスタントが自然言語コマンドを通じて Dot. デバイスと連携できるようにするオープンソースツールです。API リクエストを手動で書く必要はありません。AI にやりたいことを伝えるだけです。

## 機能

- **プラグアンドプレイ** - インストール後、自然言語でデバイスを制御
- **テキスト送信** - デバイスに指定したテキスト内容を表示
- **画像送信** - 画像をデバイスに送信して表示
- **Canvas 送信** - data と windowData を送信してカスタム画面を描画
- **タスク別名** - テキスト、画像、Canvas の内容に識別しやすいタスク名を設定
- **状態確認** - デバイスのバッテリー残量、ネットワーク状態などを確認
- **コンテンツ切り替え** - 次のコンテンツに切り替え
- **ヘルパースクリプト** - 自動化用の Python スクリプト

## 対応プラットフォーム

| プラットフォーム | 状態 | 連携方法 |
| --- | --- | --- |
| Codex | 対応済み | Codex plugin としてインストールします。このリポジトリの `.agents/plugins/marketplace.json` と `plugins/dot-skill` bundle を使用します |
| OpenAI GPT Actions | schema 経由で対応 | `openapi/dot-openapi.yaml` をインポートし、Bearer 認証を設定します |
| Claude / MCP clients | 計画中 | 現時点では OpenAPI schema を使用できます。今後 remote MCP server を追加できます |
| Cursor と skill 互換 agent | skill docs として対応済み | `skills/dot-device-openapi` と `skills/dot-canvas-designer` の一方または両方をインストールするか、このリポジトリを context として使用します |
| MCP Registry | 計画中 | Dot MCP server が用意された後に server metadata を公開します |

## インストール方法

### 方法 1: Codex plugin としてインストール

dot_skill リポジトリを Codex marketplace として追加します：

```bash
codex plugin marketplace add git@github.com:MindReset/dot_skill.git
```

次に plugin をインストールします：

```bash
codex plugin add dot-skill@mindreset-dot-skill
```

インストール後、Codex が plugin 内の skills を読み込めるように、新しい Codex スレッドを作成してください。

### 方法 2: `npx skills add` でインストール

```bash
npx skills add https://github.com/MindReset/dot_skill.git
```

デバイス操作 skill のみインストール：

```bash
npx skills add https://github.com/MindReset/dot_skill.git --skill dot-device-openapi
```

Canvas designer skill のみインストール：

```bash
npx skills add https://github.com/MindReset/dot_skill.git --skill dot-canvas-designer
```

### 方法 3: 手動インストール

```bash
mkdir -p ~/.agents/skills
ln -sfn /path/to/dot_skill/skills/dot-device-openapi ~/.agents/skills/dot-device-openapi
ln -sfn /path/to/dot_skill/skills/dot-canvas-designer ~/.agents/skills/dot-canvas-designer
```

手動インストール後、agent を再起動してください。`dot-openapi` は古いインストール向けの互換ルーターとして残っています。新規インストールでは `dot-device-openapi` と `dot-canvas-designer` の利用を推奨します。

## クイックスタート

### 環境変数の設定

```bash
export DOT_API_KEY="dot_app_your_api_key"
```

### 使用開始

インストール後、AI アシスタントに以下のように話しかけるだけ：

- "デバイス ABCD1234ABCD に '会議は午後 3 時から' と表示して"
- "デバイス ABCD1234ABCD に '会議は午後 3 時から' と表示して、会議リマインダーという名前にして"
- "デバイスの状態を確認して"
- "次のコンテンツに切り替えて"

## タスク別名

同じデバイスに複数のテキスト、画像、Canvas 内容を作成する場合、それぞれにタスク別名を付けると、デバイスのタスク一覧で用途を見分けやすくなります。Dot Skill はテキスト、画像、Canvas の送信時にこの名前を渡します。指定しない場合は既存の名前を保持し、明示的に名前の削除を依頼した場合は別名を削除します。
