## 快速指引

### 必要条件

1. 确保您已在 Dot. App 中<Link href="/docs/service/studio/api/get_api">获取并保存了 API 密钥</Link>；
2. 确保您至少有一台所属于自己的设备并已<Link href="/docs/service/studio/api/get_device_id">获取设备序列号</Link>；
3. 确保您的设备已接入电源；
4. 确保您的设备已接入网络；
5. 确保已从 Dot. App 内容工坊中添加 `文本 API` 内容到设备**循环**任务中。

<Callout type="info" title="文本 API 的用途">
	文本 API 适合把提醒、灵感、签名和小图标组合成一张适合设备阅读的文字卡片。您可以用它发送自动化消息、轻量通知、每日摘录或需要立即看到的一段文本。
</Callout>

<Callout type="info" title="关于 API 显示时机">
	API 的即时显示能力依赖设备当前是否在线。若设备使用电池供电并处于睡眠状态，内容会先保存至服务器，并在设备下一次自动唤醒后显示。您可以阅读<Link href="/docs/quote_0/faq/refresh_timing">时间机制</Link>了解更多。
</Callout>

<Callout type="info" title="关于图标输入">
	`icon` 字段支持传递 PNG Base64 数据，也支持传递完整的在线 `http(s)` 图片 URL。远程图标链接必须可匿名访问、无需鉴权或特殊 Referer，并直接返回 `image/*` 内容。
</Callout>

<Callout type="info" title="可用于调试的测试图标链接">
	以下链接当前可直接匿名访问，并返回可用于 Text API `icon` 的图片内容：

	- `https://placehold.co/40x40/png`
	- `https://httpbin.org/image/png`

	如果您想先验证自己的图标链接是否可用，可以先执行：

	```bash
	curl -I 'https://your-icon-url'
	```

	通常需要确认响应中至少满足两点：

	- HTTP 状态码为 `200`
	- `Content-Type` 为 `image/*`，例如 `image/png` 或 `image/jpeg`
</Callout>

### 文本中的换行与制表符

Text API 会保留 `message` 中的换行和制表符。

- 在 JSON 请求体中，使用 `\n` 表示换行，使用 `\t` 表示制表符。
- 如果您的 JSON 外面还包了一层 shell、JavaScript、Python、模板引擎或工具配置字符串，通常需要在外层写成 `\\n` 和 `\\t`，让最终发出的 JSON 仍然包含 `\n` 和 `\t`。
- 如果您希望设备显示字面量字符 `\n` 或 `\t`，请让最终 JSON 载荷中包含 `\\n` 或 `\\t`。

```json
{ "message": "第一行\n\t缩进后的第二行" }
{ "message": "显示字面量 \\n 和 \\t" }
```

### 响应说明

| code  | 含义             | 描述                                           |
| ----- | ---------------- | ---------------------------------------------- |
| `200` | 成功             | 设备文本 API 内容已切换/数据已更新但未切换内容 |
| `400` | 参数错误         | 无效的图标格式                                 |
| `403` | 权限不足         | 您没有权限操作此设备                           |
| `404` | 设备或内容不存在 | 设备不存在或未注册<br/>未添加文本 API 内容到循环任务中 |
| `500` | 设备响应失败     | 设备文本 API 内容切换失败                      |
