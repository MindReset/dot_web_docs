API は、異なるソフトウェア同士が通信し、データをやり取りするための「橋渡し」です。簡単に言えば、API は「通訳」や「ウェイター」のような存在で、異なるプログラム同士が「会話」できるようにします。

## API はどのように動くの？

<Steps>
  <Step>
### リクエストを送る

「デバイスに『今日は天気がいい』と表示して」

  </Step>
    <Step>
### API がリクエストを処理

API はリクエストを受け取ると、ユーザー認証とデバイス認証を行い、正当なユーザーとデバイスからの要求かを確認します。

  </Step>
    <Step>
### デバイスが操作を実行

デバイスが指示を受け、画面にテキストを表示します。

  </Step>
    <Step>
### 結果を返す

「操作成功。内容をデバイスに表示しました」

  </Step>
</Steps>

そのため Dot. API を使うと、次のようなことが可能になります。

- 重要なメール受信時、件名と一部内容を自動でデバイスに表示
- 認証コード（SMS）を受け取ったら、自動でデバイスに表示
- 玄関ベルが鳴ったら、デバイスに訪問者情報を表示
- 高度にパーソナライズされたレイアウト（個人用ダッシュボードなど）を表示
- 面白い画像や絵文字を表示

## 誰が開発者プラットフォームを使える？

1. ギーク／愛好家：ノーコードツール（ショートカット、IFTTT、Zapier など）で簡単な自動化を作る
2. プログラマー：コードを書いて複雑な自動化システムを構築する
3. 企業ユーザー：Dot. を既存の業務システムに統合し、作業効率を高める
4. IoT デバイスや自動化に興味のある方
